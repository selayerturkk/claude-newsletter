# The Claude Dispatch

An automated twice-weekly newsletter covering Anthropic/Claude news and broader tech, generated by Claude Code and delivered to your inbox.

## How it works

1. **Cron** triggers `run.sh` every Monday and Wednesday at 8:00 AM
2. `run.sh` checks `newsletters/index.json` to find the last issue date and number
3. It invokes **Claude Code** (`claude -p`) with a prompt to search the web for recent news, compose the newsletter as styled HTML, save it, and update the index
4. `send_email.py` sends the generated HTML email via Gmail SMTP
5. Every issue is archived in `newsletters/` for future reference

No API key is needed — it uses your installed Claude Code CLI directly.

## Quick start

### 1. Set up credentials

```bash
cd ~/Desktop/claude-newsletter
cp .env.example .env
```

Edit `.env` and fill in:

- **GMAIL_ADDRESS** — the Gmail account that will send the newsletter
- **GMAIL_APP_PASSWORD** — a Gmail App Password (see below)
- **RECIPIENT_EMAIL** — where to deliver the newsletter

### 2. Create a Gmail App Password

1. Go to [Google Account Security](https://myaccount.google.com/security)
2. Enable **2-Step Verification** if you haven't already
3. Go to [App Passwords](https://myaccount.google.com/apppasswords)
4. Select **Mail** and **Other (Custom name)**, name it "Claude Dispatch"
5. Copy the 16-character password and paste it into `.env` (no spaces)

### 3. Test it manually

```bash
# Make the scripts executable
chmod +x run.sh install_cron.sh

# Generate and send a test newsletter
./run.sh
```

This will generate Issue #1, save it to `newsletters/`, and email it to you. Check your inbox (and spam folder the first time).

To generate without sending email, you can run just the Claude Code step:

```bash
claude -p "Generate Issue #1 of The Claude Dispatch for $(date +%Y-%m-%d). Cover news from the past 7 days. Follow ALL instructions in CLAUDE.md exactly." --allowedTools "WebSearch,WebFetch,Read,Write,Glob,Grep,Edit"
```

To send a specific issue without regenerating:

```bash
python3 send_email.py newsletters/2026-05-22-newsletter.html
```

### 4. Install the cron job

```bash
./install_cron.sh
```

This installs a cron entry: `0 8 * * 1,3` (8:00 AM every Monday and Wednesday).

Verify with `crontab -l`. Remove with `crontab -e`.

## Changing the schedule or recipient

**Schedule:** Edit `install_cron.sh` and change the `CRON_SCHEDULE` variable, then re-run it. Common patterns:

| Schedule | Cron expression |
|---|---|
| Mon & Wed 8 AM | `0 8 * * 1,3` |
| Every weekday 9 AM | `0 9 * * 1-5` |
| Every Sunday 10 AM | `0 10 * * 0` |
| Daily 7:30 AM | `30 7 * * *` |

**Recipient:** Change `RECIPIENT_EMAIL` in `.env`.

**Multiple recipients:** Set `RECIPIENT_EMAIL` to a comma-separated list (e.g., `a@gmail.com,b@gmail.com`). Note: some email clients may show all recipients.

## Important: cron only runs when your Mac is awake

macOS cron jobs only execute while your machine is powered on and awake. If your Mac is asleep or off at 8 AM, the job will **not** run and will **not** retry.

### If you miss an issue

Just run `./run.sh` manually. The script automatically detects the gap since the last issue and covers the right date range.

### If you want more reliable scheduling

Options, from simplest to most robust:

1. **Keep your Mac awake** — use `caffeinate` or set Energy Saver to prevent sleep during newsletter hours.

2. **Use `launchd` instead of cron** — macOS's native scheduler can run missed jobs when the machine wakes up. Create a plist in `~/Library/LaunchAgents/`. See Apple's documentation on `launchd`.

3. **Use a cloud server** — run the script on a small VPS (e.g., a free-tier EC2 instance, a $5/mo DigitalOcean droplet, or a Raspberry Pi). Install Claude Code on it and set up cron there. This is the most reliable option.

4. **Use GitHub Actions** — set up a scheduled workflow that runs the script. You'd need to store credentials as GitHub Secrets and install Claude Code in the runner.

## File structure

```
claude-newsletter/
├── CLAUDE.md              # Instructions Claude Code follows when generating
├── run.sh                 # Main entry point (cron calls this)
├── send_email.py          # Gmail SMTP sender
├── install_cron.sh        # One-command cron setup
├── .env                   # Your credentials (gitignored)
├── .env.example           # Template showing what to fill in
├── .gitignore
├── requirements.txt
├── run.log                # Execution log (gitignored)
├── README.md
└── newsletters/
    ├── index.json         # Log of all issues (number, date, filename, subject)
    └── YYYY-MM-DD-newsletter.html  # Archived issues
```

## Logs

Each run appends to `run.log`. Check it for debugging:

```bash
tail -50 run.log
```

## Dependencies

- **Claude Code** CLI (`claude`) — must be installed and authenticated
- **Python 3.9+** — for the email sender (uses only standard library)
- No pip packages required
