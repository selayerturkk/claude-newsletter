Understanding Self-Hosted Sandboxes

Self-hosted sandboxesSecure environments where companies can run AI agents on their own infrastructure represent a crucial step in enterprise AI adoption. Here's what they actually do and why they matter:

What they solve: Traditional cloud-based AI agents execute code and access data on the provider's infrastructure. For enterprises with sensitive data or strict compliance requirements, this creates security and governance challenges.

How they work: With self-hosted sandboxes, you keep sensitive files, packages, and services in your own infrastructure while the agent loop that handles orchestration stays on Anthropic's infrastructure. Inside your perimeter, network policies, audit logging, and security tooling are already in place, and files and repositories don't leave.

Why it matters: This hybrid approach lets enterprises get the benefits of advanced AI agents without compromising on data security or compliance. You control compute resources, runtime environments, and data access while still leveraging Anthropic's agent orchestration capabilities.

Getting started: You can bring any sandbox client or start with supported providers like Cloudflare, which runs sandboxes at scale using microVMs. Anthropic provides docs, cookbooks for setup, and deployment tools in the Claude Console.
